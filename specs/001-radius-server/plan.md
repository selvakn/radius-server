# Implementation Plan: RADIUS Server

**Branch**: `001-radius-server` | **Date**: 2026-06-22 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/001-radius-server/spec.md`

## Summary

Build a self-contained RADIUS authentication server for PPPoE dial-up users (MikroTik-compatible), distributed as a single Go binary. Uses `layeh.com/radius` for RFC 2865 protocol handling, `modernc.org/sqlite` (pure Go, no CGO) for user persistence, and a minimal server-side-rendered HTML admin UI over HTTP. Admin accounts are defined in a YAML config file with bcrypt-hashed passwords.

## Technical Context

**Language/Version**: Go 1.22+  
**Primary Dependencies**: `layeh.com/radius`, `glebarez/sqlite` (modernc.org/sqlite wrapper), `github.com/go-chi/chi/v5`, `gopkg.in/yaml.v3`, `golang.org/x/crypto/bcrypt`, `github.com/pressly/goose/v3`  
**Storage**: SQLite via modernc.org/sqlite (pure Go, CGO-free)  
**Testing**: `go test ./...` with table-driven tests; coverage gate ≥70%  
**Target Platform**: Linux amd64 (single static binary); developed on any OS  
**Project Type**: Server daemon (RADIUS + HTTP)  
**Performance Goals**: <500ms RADIUS response; 1,000 concurrent auth requests  
**Constraints**: Single binary, no external runtime dependencies, <100 dial-up users  
**Scale/Scope**: Small site — ≤100 users, low auth concurrency

### Phase 4: RADIUS Authentication

**Package**: `internal/auth`

**Implementation**:
- `Handler` struct holds `*db.DB` and `shared_secret string`
- `Handler.ServeRADIUS(w radius.ResponseWriter, r *radius.Request)` — main handler
- `Handler.authenticate(req *radius.Request) (*db.User, error)` — orchestrates lookup and verification
- `pap.go`: `VerifyPAP(req *radius.Request, secret, passwordHash string) bool`
- On accept: add `Framed-Protocol`, `Framed-Compression`, and `Mikrotik-Rate-Limit` VSA if rates set
- On reject: add `Reply-Message` attribute

**MikroTik VSA encoding**:
```go
// Vendor 14988, Attribute 8, format: "2048k/1024k"
func mikrotikRateLimit(down, up int) radius.Attribute
```

**Error handling**:
- Malformed packet → log warning, no response (layeh/radius handles this)
- DB error → Access-Reject (fail-safe)

**Tests**: PAP accept, PAP reject (wrong password), disabled user reject, user not found reject, rate-limit attribute in accept, DB error → reject.

**Commit**: `feat: implement RADIUS authentication handler with PAP and MikroTik VSA`

### Phase 5: Admin Web UI

**Package**: `internal/web`

**Session store** (`session.go`):
- `Store` struct: `sync.RWMutex` + `map[string]Session`
- `Store.Create(adminUsername string) string` — returns session token
- `Store.Get(token string) (*Session, bool)`
- `Store.Delete(token string)`
- Background goroutine to expire sessions after 8h

**HTTP server** (`server.go`):
- chi router
- Middleware: session validation on protected routes, CSRF validation on POST
- Static file embedding via `//go:embed templates/*`

**Handlers** (`handlers.go`):
- `GET /login` → render login form
- `POST /login` → validate credentials, set cookie, redirect
- `POST /logout` → clear session
- `GET /` → list users (table)
- `GET /users/new` → new user form
- `POST /users` → create user
- `GET /users/{id}/edit` → edit form
- `POST /users/{id}` → update user
- `POST /users/{id}/disable` → disable
- `POST /users/{id}/enable` → enable
- `POST /users/{id}/delete` → delete

**Templates** (HTML, `html/template`):
- `layout.html`: base layout, flash message slot
- `login.html`: username + password form, CSRF token
- `users.html`: table with per-row actions, new user link

**Tests**: Login success, login failure, session middleware, CSRF validation, user create/disable/delete handlers.

**Commit**: `feat: implement admin web UI with chi router and server-side sessions`

**Package**: `cmd/server`
